# Configuration & Initialization

This document defines the `pulse init` interactive flow and the `.pulse-config.yaml` schema. It is loaded only when the user invokes `pulse init`.

---

## `pulse init` Interactive Flow

Run `pulse init` to set up Pulse in an Obsidian vault. The flow consists of 6 sequential steps. If a step has a default value, pressing Enter without input accepts the default.

### Step 1: Confirm Vault Path

Auto-detect the Obsidian vault by scanning common paths:

1. Check `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/*/` for directories containing a `.obsidian/` subfolder.
2. If exactly one vault is found, present it as the default and ask for confirmation.
3. If multiple vaults are found, list them numbered and ask the user to select one.
4. If no vault is found, prompt the user to enter the full path manually.

**Prompt:**
```
Detected Obsidian vault: /Users/charliec/Library/Mobile Documents/iCloud~md~obsidian/Documents/MyVault
Use this vault? [Y/n]:
```
Or if none detected:
```
No Obsidian vault detected automatically.
Enter the full path to your Obsidian vault:
```

**Validation:** The path must exist and contain a `.obsidian/` directory. If validation fails, ask the user to re-enter.

### Step 2: Confirm Pulse Directory Name

Ask the user for the directory name where Pulse will store its generated reports inside the vault.

- **Default:** `00. Pulse`

**Prompt:**
```
Pulse directory name [00. Pulse]:
```

The directory will be created at `{vault_path}/{pulse_dir}/` if it does not already exist.

### Step 3: Create Directory Structure

Automatically create the following subdirectories inside the Pulse directory:

```
{vault_path}/{pulse_dir}/
  10. Daily/
  20. Weekly/
  30. Monthly/
  40. Annual/
```

- If directories already exist, skip creation silently.
- Print a summary of created directories.

**Output:**
```
Created directory structure:
  ✓ 00. Pulse/10. Daily/
  ✓ 00. Pulse/20. Weekly/
  ✓ 00. Pulse/30. Monthly/
  ✓ 00. Pulse/40. Annual/
```

### Step 4: Configure Data Source Directories

For each of the 8 dimensions, ask the user to specify one or more source directories within the vault. These directories will be scanned for files matching date patterns during report generation.

**The 8 dimensions:**
1. `daily-review` — Daily check / review notes
2. `daily-plan` — Daily planning notes
3. `weekly-review` — Weekly review notes
4. `weekly-plan` — Weekly planning notes
5. `monthly-review` — Monthly review notes
6. `monthly-plan` — Monthly planning notes
7. `annual-review` — Annual review notes
8. `annual-plan` — Annual planning notes

**Prompt per dimension:**
```
[daily-review] Enter source directories (comma-separated, relative to vault root).
Press Enter to skip:
```

- The user can enter one or more directory paths separated by commas (e.g., `Journal/Daily, Work/Logs`).
- Pressing Enter without input skips that dimension (sets `dirs: []`).
- All paths are relative to `vault_path`.
- Validate that entered directories exist in the vault. Warn (but do not block) if a directory does not exist yet.
- The user can always add or modify sources later by editing `.pulse-config.yaml` directly.

### Step 5: Set Timezone and Language

Ask the user for their timezone and preferred language.

- **Timezone default:** `Asia/Shanghai`
- **Language default:** `zh-CN`

**Prompts:**
```
Timezone [Asia/Shanghai]:
Language [zh-CN]:
```

Supported language values: `zh-CN`, `en-US`. The language setting controls the language of generated report content.

### Step 6: Generate `.pulse-config.yaml`

Write the configuration file to `{vault_path}/{pulse_dir}/.pulse-config.yaml`.

- If the file already exists, ask for confirmation before overwriting.
- Print the full path of the generated config file upon success.

**Output:**
```
Configuration saved to: /path/to/vault/00. Pulse/.pulse-config.yaml
Pulse is ready. Run `pulse daily` to generate your first daily report.
```

## Config File Schema

The generated `.pulse-config.yaml` follows this schema:

```yaml
# Pulse PDCA System Configuration
# Generated by: pulse init
# Last updated: {date}

vault_path: "/path/to/obsidian/vault"
pulse_dir: "00. Pulse"

sources:
  daily-review:
    # Match: filename contains YYYY-MM-DD where date = target date
    dirs: []
  daily-plan:
    dirs: []
  weekly-review:
    # Match: filename contains YYYY-MM-DD where date falls within target Mon-Sun week
    dirs: []
  weekly-plan:
    dirs: []
  monthly-review:
    # Match: filename contains YYYY-MM where month = target month
    dirs: []
  monthly-plan:
    dirs: []
  annual-review:
    # Match: filename contains YYYY where year = target year
    dirs: []
  annual-plan:
    dirs: []

timezone: "Asia/Shanghai"
language: "zh-CN"
```

### Field Descriptions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `vault_path` | string | Yes | Absolute path to the Obsidian vault root directory |
| `pulse_dir` | string | Yes | Name of the Pulse directory inside the vault |
| `sources` | object | Yes | Map of 8 dimensions to their source directory configurations |
| `sources.<dimension>.dirs` | list[string] | Yes | List of directories (relative to `vault_path`) to scan for source files |
| `timezone` | string | Yes | IANA timezone identifier for date calculations |
| `language` | string | Yes | Language code for generated report content (`zh-CN` or `en-US`) |

## Date Matching Rules

When scanning source directories for files, Pulse matches filenames against date patterns based on the dimension type:

| Dimension | Pattern | Example |
|-----------|---------|---------|
| `daily-*` | Filename contains `YYYY-MM-DD` or `YYYY年MM月DD日`, date = target day | `2026-02-21-meeting-notes.md` or `2026年02月21日总结.md` matches daily-review for 2026-02-21 |
| `weekly-*` | Filename contains `YYYY-MM-DD` or `YYYY年MM月DD日`, date within target Mon-Sun | Files dated 2026-02-16 to 2026-02-22 match W08 |
| `monthly-*` | Filename contains `YYYY-MM` or `YYYY年MM月`, month = target | `2026-02-report.md` or `2026年02月总结.md` matches February |
| `annual-*` | Filename contains `YYYY`, year = target | `2026-goals.md` matches 2026 |

### Matching Details

- **Daily dimensions** (`daily-review`, `daily-plan`): Try TWO patterns in order:
  1. Standard format: Extract `YYYY-MM-DD` from the filename using the regex `\d{4}-\d{2}-\d{2}`.
  2. Chinese format: Extract date from the filename using the regex `(\d{4})年(\d{2})月(\d{2})日`, then normalize to `YYYY-MM-DD`.
  The extracted date (from either pattern) must equal the target date exactly.
- **Weekly dimensions** (`weekly-review`, `weekly-plan`): Extract date using the same two patterns as daily dimensions. The extracted date must fall within the target ISO week (Monday through Sunday). Week numbers follow ISO 8601 (weeks start on Monday).
- **Monthly dimensions** (`monthly-review`, `monthly-plan`): Try TWO patterns:
  1. Standard: Extract `YYYY-MM` using the regex `\d{4}-\d{2}`.
  2. Chinese: Extract year-month using the regex `(\d{4})年(\d{2})月`, then normalize to `YYYY-MM`.
  The extracted year-month must match the target month. Note: if a filename contains a full date (either format), the year-month portion is extracted from it.
- **Annual dimensions** (`annual-review`, `annual-plan`): Extract `YYYY` from the filename using the regex `\d{4}`. The extracted year must match the target year. To avoid false positives, only match 4-digit sequences that are plausible years (e.g., 2000-2099).
- **File extension filter**: Only `.md` files are scanned.
- **Subdirectory scanning**: Source directories are scanned recursively — files in subdirectories are included.
- **Multiple matches**: All files that match the pattern are collected and provided as input to the routine.

## How to Read Config at Runtime

When any Pulse command runs (e.g., `pulse daily`, `pulse weekly`), the configuration is loaded as follows:

1. **Locate config file**: Parse YAML from `{vault_path}/{pulse_dir}/.pulse-config.yaml`. The `vault_path` can be determined by:
   - Checking the current working directory for a `.pulse-config.yaml` up the directory tree, or
   - Using a `--vault` flag if provided by the user.

2. **Resolve source directories**: All paths in `sources.<dimension>.dirs` are relative to `vault_path`. Resolve full paths by joining:
   ```
   {vault_path}/{source_dir}
   ```
   For example, if `vault_path` is `/Users/charliec/Vault` and a `daily-review` dir is `Journal/Daily`, the resolved path is `/Users/charliec/Vault/Journal/Daily`.

3. **Scan for matching files**: For each resolved directory, recursively scan for `.md` files whose filenames match the date pattern for the current dimension (see Date Matching Rules above).

4. **Collect input**: Gather all matched files as the input file list for the current routine. Pass this list to the LLM along with the appropriate template for report generation.

5. **Determine output path**: Based on the command, write the generated report to the corresponding subdirectory:
   - `pulse daily` -> `{vault_path}/{pulse_dir}/10. Daily/`
   - `pulse weekly` -> `{vault_path}/{pulse_dir}/20. Weekly/`
   - `pulse monthly` -> `{vault_path}/{pulse_dir}/30. Monthly/`
   - `pulse annual` -> `{vault_path}/{pulse_dir}/40. Annual/`

## Editing Config

Users can edit `.pulse-config.yaml` directly in any text editor (including Obsidian) to add, remove, or modify data source directories without re-running `pulse init`.

### Common Edits

**Add a new source directory:**
```yaml
sources:
  daily-review:
    dirs:
      - "Journal/Daily"
      - "Work/Standup"    # newly added
```

**Remove a source directory:**
Simply delete the line from the `dirs` list.

**Change timezone:**
```yaml
timezone: "America/New_York"
```

**Change language:**
```yaml
language: "en-US"
```

### Validation

Pulse validates the config file on each run:
- `vault_path` must be an existing directory.
- `pulse_dir` must be a non-empty string.
- Each entry in `dirs` should be a valid relative path. Non-existent directories trigger a warning but do not halt execution.
- `timezone` must be a valid IANA timezone identifier.
- `language` must be one of the supported values (`zh-CN`, `en-US`).

If `.pulse-config.yaml` is missing or invalid, Pulse prints an error message and suggests running `pulse init`.
