# PRD 修改流程

## 第一步：理解修改上下文

1. **读取目标文档**：完整读取用户指定的 PRD 文档
2. **收集用户口头指令**：记录用户在对话中提出的修改要求
3. **扫描文档内嵌指令**：在文档中搜索所有 `{{{` 和 `}}}` 包裹的修改指令
   - 内嵌指令格式：`{{{修改说明文字}}}`
   - 记录每条指令的位置（所在章节/上下文）和具体内容

## 第二步：生成修改计划

将所有修改指令（口头 + 内嵌）整理为结构化的修改计划：

```markdown
# PRD 修改计划

## 修改来源
- 用户口头指令：X 条
- 文档内嵌指令：Y 条

## 修改清单

### 修改 1：[修改标题]
- **来源**：口头指令 / 内嵌指令（第N章节）
- **原始指令**：[用户原话或内嵌内容]
- **修改位置**：[具体章节/段落]
- **修改内容**：[具体要做什么]
```

**整理原则**：
- 如果多条指令涉及同一区域，合并为一条修改
- 按文档顺序排列修改清单
- 对模糊指令，在计划中标注需要向用户确认的问题

向用户展示修改计划，确认后再执行。

## 第三步：逐条执行修改

按修改计划逐条执行：
- 每条修改完成后，简要说明改动内容
- 修改时遵循文档现有的格式和风格
- 新增内容参考 `references/PRD-TEMPLATE.md` 的结构规范
- 如果修改涉及章节编号调整，确保全文编号连续

## 第四步：清理与校验

1. **清除内嵌指令**：删除文档中所有已执行的 `{{{...}}}` 标记
2. **一致性检查**：确认修改后文档内部逻辑一致（如编号、交叉引用）
3. **输出修改摘要**：

```markdown
# PRD 修改完成报告

## 已完成修改
1. [修改1简述]
2. [修改2简述]

## 需注意事项
- [如有需要用户进一步确认的内容]
```
